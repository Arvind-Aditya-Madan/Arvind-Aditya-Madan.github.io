ACC Spread Predictor

A second-half ACC men’s basketball spread prediction project built to forecast home-minus-away point spread for remaining games after February 6 using only information available from the first half of the season.

Submitted for the Triangle Sports Analytics Competition, placing 6th.

Challenge

The task was to predict the point spread for all remaining ACC men’s basketball games starting on February 6. The model needed to behave like a real deployment system: train on historical seasons, build features only from first-half games, and predict second-half games without leakage.

Data construction

Data pipeline

Historical game-level data from 2014–2025 was collected in R using ncaahoopR for ACC teams, with box scores appended. The project moved to Python for feature engineering, modeling, evaluation, and prediction generation.

Target

The primary target was spread = home score − away score. Secondary explorations included home score, away score, and total score, but final evaluation focused on spread prediction.

Temporal design

Midseason was defined as a calendar date rather than a game count. All games before the midseason cutoff were used to construct team-season features. Second-half games were stripped down to date, teams, home/away designation, and target spread before first-half team features were merged in.

Feature engineering

Team strength

Offensive rating, defensive rating, net rating, pace, shooting efficiency, turnover rate, free throw rate, rebounding, assists, fouls, win percentage, and margin-based measures.

Context

Rest days, days since midseason, AP Poll ranking indicators, ranked-opponent performance, ACC-only performance, last-five-game form, and venue-adjusted home/away ratings.

RAW vs. DIFF features

The project tested RAW home/away features, DIFF features, BOTH, and HYBRID representations. DIFF features — home statistic minus away statistic — consistently outperformed RAW features because the representation aligned directly with the spread definition.

Prediction intervals

In addition to point predictions, the project built split conformal prediction intervals for spread forecasts. For each test season, the model was fit on earlier seasons, calibrated on the most recent prior season, and evaluated on the test season. Absolute calibration residuals determined the conformal radius q, producing intervals of [prediction − q, prediction + q].

A nominal coverage level of 75% was used to stay above the target of 70% empirical coverage while keeping interval widths manageable.

Final feature set

The final model used DIFF-based features covering efficiency, form, ACC-only performance, venue-adjusted ratings, rest effects, ranking context, and performance against ranked opponents. Team identifiers were included to allow the model to capture program-level persistence and coaching/system effects.

What I learned

Leakage control matters

Strict temporal separation made the project closer to a real forecasting workflow and prevented second-half information from entering first-half features.

Representation matters

DIFF features were more stable and predictive than RAW home/away features because the target itself was a home-away difference.

Uncertainty matters

Prediction intervals gave a more honest picture of spread uncertainty than point predictions alone.

Sports models are noisy

Even with strong features, lineup variation, injuries, home/away context, and opponent dynamics create uncertainty that the model must account for.
